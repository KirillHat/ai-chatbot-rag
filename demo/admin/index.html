How it works under the hood
  • Bytes are SHA-256'd — re-uploading the same file is a no-op.
  • Loader picks a parser by extension (pypdf for PDF; UTF-8 / Latin-1 fallback for text).
  • Recursive splitter cuts on paragraphs → sentences → words → chars to keep chunks coherent.
  • Each chunk is embedded with the default ONNX MiniLM-L6-v2 model (CPU, no API key).
  • Vectors land in ChromaDB; metadata (name, char count) goes to SQLite.
  • The next /api/chat retrieves the top-K chunks and Claude answers from them with citations.
